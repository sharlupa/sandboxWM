# sandboxWM

[English version](./README_EN.md)

**sandboxWM** — экспериментальный Wayland-композитор на Rust: обычный BSP/Dwindle-тайлинг можно переключить в физическую песочницу, где окна имеют массу, падают, сталкиваются и вращаются.

> ⚠️ **Статус: ранняя версия (0.1.0).** Это рабочий прототип. Рисование линий, сохранение сессии, связи между окнами, деформация и слайсинг пока описывают будущее направление проекта, а не готовые возможности.

## Что уже работает

- Wayland-композитор на **Smithay 0.7**: `xdg_shell`, SHM, seat/input, output, серверные декорации и буфер обмена.
- Два режима запуска:
  - вложенное окно **Winit** в действующей графической сессии;
  - нативный **DRM/KMS**-режим из TTY с libseat, GBM/EGL, udev и libinput.
- Тайловый менеджер BSP/Dwindle: автоматическое разбиение, фокусировка, изменение пропорций мышью.
- Физический режим на **rapier2d**: гравитация, широкий статический пол, столкновения, damping, CCD, синхронизация размеров коллайдеров, drag без телепортации и камера на бесконечном холсте.
- Поворот окон: отрисовка GLES с трансформацией текстуры, hit-test с учётом угла и удержание `Super+A` / `Super+D`.
- Рендер по требованию, DMA-BUF для импорта клиентских буферов, программный курсор в DRM-режиме.
- Базовые протоколы `wlr-screencopy` (захват экрана) и `wlr-output-management` (пока только чтение состояния выходов).
- Поддержка X11-приложений через **XWayland** (rootless): синхронизация стека окон и фокуса с клиентами вроде Steam, корректная передача событий указателя (клики, перемещение, прокрутка) в X11/CEF-клиенты.
- Визуальная линия физического пола в DRM-режиме. Декоративная сетка точек была убрана: сотни отдельных render-элементов у пола вызывали микрофризы.

## Управление

| Комбинация | Действие |
|---|---|
| `Super+Enter` | Запустить терминал (по умолчанию `kitty`) |
| `Super+Q` / `Super+Esc` | Завершить WM |
| `Super+W` | Закрыть активное окно |
| `Super+G` | Тайлинг ↔ физический режим |
| `Super+A` / `Super+D` | Удерживать для поворота активного окна в физическом режиме |
| `Super+Стрелки` | Тайлинг: смена фокуса; физика: движение камеры |
| `Super+-` / `Super+=` | Физика: отдалить / приблизить камеру |
| `Super+0` | Физика: сбросить позицию и масштаб камеры |
| `Super+колесо` | Физика: масштабировать камеру (только при удержании Super) |
| Средняя кнопка мыши + drag | Физика: перемещать камеру |
| `Super+ЛКМ` + drag | Тайлинг: изменить пропорции; физика: тащить окно |
| `ЛКМ` | Фокус и обычное взаимодействие с окном |
| `Ctrl+Alt+F1…F12` | Переключить VT в TTY/DRM-режиме |

## Сборка и запуск

Нужны Rust, системные библиотеки Wayland/DRM/GBM/EGL и libseat для нативного режима.



Режим выбирается автоматически: если нет `WAYLAND_DISPLAY` и `DISPLAY`, запускается DRM/KMS; иначе — Winit.

## Конфигурация

При старте читается первая найденная директория:

1. `$SANDBOXWM_CONFIG`
2. `$XDG_CONFIG_HOME/sandboxWM/`
3. `~/.config/sandboxWM/`
4. `./config/` из корня репозитория

Все файлы `*.conf` в этой директории объединяются как TOML. Сейчас реально применяются `[physics]` и `[controls]` из `config/sandboxWM.conf`. Остальные секции (`drawing`, `materials`, `joints`, `soft_body`, `slicing`, `session`, `window_props`) уже имеют схему и примеры конфигурации, но пока не подключены к логике.

## Структура

- `src/main.rs` — вход, выбор бэкенда и основной цикл.
- `src/backend_drm.rs` — DRM/KMS, EGL/GBM, libinput, курсор, пол и screencopy.
- `src/state.rs` — глобальное состояние, раскладка, физика, камера и Wayland handlers.
- `src/input.rs` — клавиатура, мышь и горячие клавиши.
- `src/xwm.rs` — интеграция XWayland (X11 window manager): синхронизация стека окон и фокуса для X11-клиентов.
- `src/physics.rs` — обёртка rapier2d.
- `src/render.rs` — GLES-элементы и поворот окон.
- `src/tiling.rs` — BSP/Dwindle-дерево.
- `src/screencopy.rs`, `src/output_manager.rs` — wlr-протоколы.
- `config/` — модульные TOML-конфиги.
